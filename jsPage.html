JavaScript Evolution and Purpose

JavaScript (JS) is a scripting language that is known as the three, core languages to build web applications and was created in 1995 by Brendan Eich during his time at Netscape. It has gained a lot of popularity over the years and has added functionality. It has allowed uses to interact with the content to make the experience more enjoyable. JS works by executing functions on the client side but the introduction of Node.js has enabled to be executed on servers. These functions are prototype based which means it's a multiparadigm language. JavaScript supports object-oriented programming, imperative and functional programming styles (cited from style ).





Purpose & Uses

Computers used to be much slower and browsers and experienced lag during web browsing on servers. Along with the expense of hosting files, this led most web developers to believe that JavaScript was a terrible idea. JavaScript changed the definition of web browsing and is executed on the user's computer when they access the page (client-side). This made sites more responsive on the user end and provided less traffic and processing strain on the server. Nowadays, computers are much faster and bandwidth is considered to be less cheaper which negates the downsides of JavaScript.

JavaScript can be used in a various scenario but is commonly utilised as a front-end web development tool. Some of uses and benefits include:

  • Website Development and Applications
  • Server Applications using Node.js (applications distributed efficiently over the systems with the help of a server)
  • Graphics on webpages including 2D and 3D shapes can be displayed (pie-chart shown under the COVID-19 Tracker)
  • Supports a lot of frameworks such as JQuery, Node, Angular, React

Brief History of JavaScript

The decade of 1990 was an important time for the internet to flourish. While Netscape (Netscape Navigator) and Microsoft (Internet Explorer) were going head to head with their browsers, Brendan developed the scripting language, Mocha, in 10 days. This immediately became known as LiveScript and later, JavaScript.

JavaScript rapidly grew which meant that it needed to be correctly maintained and developed. For this, Netscape assigned an organisation to standardise computing known European Computer Manufacturers Association ( ECMA ) in 1995. The ECMA specifications were labelled ECMA-262 and ECMAScript languages included JavaScript, JScript, and ActionScript.

These first ECMA script was released in 1997 included many specifications such as generators, iterators and destructuring assignments but a year later ECMA Script (ES) 2 and 3 were introduced. ECMA3 was introduced in 2000 added features such as regular expression, exception handling features and strict Boolean comparison operator ('==='). This opeator has been used to in CODVID-19 Tracker when we compare the country typed to the country data in the JSON file.

This quickly changed as we Jesse Garett's paper about Ajax in 2005 about how it dramatically improved user experience by allowing web pages to feel easier to navigate. This paper is one of the founding backbones of the JavaScript community. The community responded with large and popular JavaScript frameworks and libraries, such as Dojo and Mootools and jQuery.

ES4 was abandoned and ES3 was renamed to ES5 in 2009 which included features such as parsing JSON files (shown in COVID Tracker). During this time, Node.js was another framework environment enabling JavaScript to be executed without a browser.

In 2015, ES6 was released but was later, renamed to ES2015. This naming pattern still exists today for latest releases of java script standard. This introduced to let and const variable types which is made it easier for to track the names and values of different variables.

Today, the latest script we are currently using is ES2019 released in June 2019. New features such as asynchronous iterators, asynchronous generators, and new regular expression have been introduced. JavaScript supports a variety of frameworks and is still currently being used in tech-leading companies such as Google, Facebook, Microsoft and many more.

COVID-19 Tracker

This application allows the user to input a country of their choice and outputs a visualisation regarding the current pandemic situation in real time. COVID-19 Tracker is achieved using Asynchronous JavaScript And XML (AJAX) and Document Object Model (DOM) manipulation. When the country that is typed is submitted, it makes to a request to the API with this API .

After it makes the HTTP Request, we receive a JSON file which contains objects of each country containing the corresponding data. This data is processed in JavaScript and accordingly plots the visualisations after the computation is completed.
